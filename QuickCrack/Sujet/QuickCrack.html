Key 2 – Reconstruct SSH key

One of the previously downloaded files (filename is random, you need to find which one is the good one in your “out/www” directory) is the SSH key* in binary format where zeros are « . » and 1 are « * ».

Every eight characters group is giving you 1 character (ASCII code).

Return files should be kept.

Example:

  • Content: .*****...*.***....*.****.*****....**..**..**.***..**.*....**...*
  • “Decoded” content:  |\/|3741

Provided example:

  • A ssh key (id_ssh.sample)

Expected result:

  • Key must be saved in « ./out/id_ssh »
Hints:
  • GREP, SED, PRINTF, READ, WHILE, FILE, GUNZIP
  • a SSH key is a file that replaced password for authentication using asymetric cryptography. It allows “automatic” connection, but in a more secured way.

Key 4 – Uncrypt the OASIS password

To crack encrypted passwords you will use a “password dictionary”, available on http://<IP>/dictpasswords.txt . Each line contains a single element.

You must try all possible combinations of elements of this file because the password is a combination of elements of the dictionary. However an element is never repeated twice. You also need to try uppercase version of element (but try only all uppercase or lowercase element, but not a mixed).

Example, if the dictionary contains:

  • word
  • verb
  • 01
  • 02

Your script must try (some examples):

  • word
  • WORDverb01
  • verbWORD0201
  • But not:
    • wORd
    • WORDword

Please note that all passwords are using the same crypto

How-to decrypt: « openssl aes-256-cbc -d -a -k <password> »
