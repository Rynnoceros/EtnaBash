Instructions

Write a hangman game in shell.

Your script must be named hangman and accept one or more files in arguments, with option --source.

Those files are words databases for your game. There are text file, each one being a word, example:

  • prepetna
  • Piscine
  • test

Your script is starting by picking up (randomly) a word in those files.

If the option --easy has been used, you are revealing the first and last letter of the mystery.

Of course options can be specified in any order.

Corrects invokations are:

./hangman.sh --source d.txt --easy --source f.txt
Copy

Incorrects invokations are (with errors that your script must display):

$> ./hangman.sh
No source have been provided!
$> ./hangman.sh --easy
No source have been provided!
$> ./hangman.sh --source
Incorrect usage!
$> ./hangman.sh --source --easy
--easy cannot be read!
Copy

Then ask user for an input, after displaying some information (see examples).

If the user enter a single letter, every occurance of the letter is discovered. If every letters have been discovered, the user wins.

If the user enter a previously enterred letter, nothing is happening (asking for another input).

If the letter is not found if the word, user is losing one try.

If the user enter a word, and the word is correct, he wins. If it's incorrect, he is losing 2 tries.

After 10 fails, he loose the game, and the word is revealed.

Words can be provided in lower or uppercase. Same for user input. But your game must not be case sensitive!

Hints
  • shift, sed, read, while, random, head, tail, wc, tr, grep, ${::}
