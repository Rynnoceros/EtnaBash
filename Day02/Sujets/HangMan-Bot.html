Instructions

Now wrote a script that can solve the game (your first bot!).

As the game script, it accepts several sources.

And, with --game option you specify the path to the game.

It starts by invoking the game and reading mistery length.

Level1 (not really a winner...)

Your scripting is just entering random letter. But never the same :)

Level2 (now, it can win!)

Your scripting is getting some IA: it checks in databases for current matches and if only one word found it proposing it.

With grep, you can use . in the search string as a wildcard: it means that any character (but exactly one) can be at the . place.

Reminder: ^ means "at the begin of the line", $ means "no content after" (end of line).

Example:

# grep '^r..t:' /etc/passwd
root:[...]
#

Level3 (we have a winner)

From databases, it decided which letter to offer, with this algorithm:

  • first, it makes a list of possible "matches" depending of the length and already discovered letters (as in level2)
  • from this list it choose the best candidate

Example 1:

mistery:		____
list is: 		tree four five dogs love
it will try "f", because 2 matches starting with a f
mistery:		b___
list is: 		bike boat bite
it will try "i" (because there is 2 matches for it)
mistery:		____
list is: 		bike boat
it will try b first (sure), and then "i" or "o" randomly...
