Instructions

Write a script that monitors all files (including all subdirectories!) in a path.

It saves a database of files and theirs checksum (fingerprint) in .redlightgreenlightdb (file format is free).

With --new parameter, the database is created.

Add a parameter --digest to specify the fingerprint algorithm to use (could be: md5, sha1, sha256, sha384, sha512), used only on database creation. Defaults is sha1.

With --check parameter, it displays file/directory modifications (respect output format of the example!):

  • N = new, D = deleted, M = modified
  • F = file, D = directory

Warning: of course your database file must not appears in the reports!

Example:
$> ./redlightgreenlight.sh --check
No database found, exiting!
$> ./redlightgreenlight.sh --new
Please wait, building database using sha1... done!
$> mkdir miaou; rm etna/passwords; touch etna/new; echo blabla > interview
$> ./redlightgreenlight.sh --check
etna/new [NF]
etna/passwords [DF]
interview [MF]
miaou [ND]
$> ./redlightgreenlight.sh --new --digest sha384
An existing database have been found, exiting!
$>
