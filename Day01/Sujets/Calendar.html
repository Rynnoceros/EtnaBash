Instructions

Your script will display a 5 days calendar starting from today if no argument is provided.

Exact expected result must be like this (if we are the september 4th...):

$> ./calendar
| Monday           | Tuesday          | Wednesday        | Thursday         | Friday
| 04 September     | 05 September     | 06 September     | 07 September     | 08 September
|------------------|------------------|------------------|------------------|------------------
| Learning shell   | Starting startup | Selling businnes | Flight to Bahama | Checking hotel
|            08:00 |            10:00 |            14:00 |            12:00 |            11:00
| C/C++ programmat |                  |                  |                  | Beach time!
|            14:00 |                  |                  |                  |            14:00
$>
Copy

Events can be added using --add argument (do not check the format of the argument, it's expected to be good):

$> ./calendar --add "2018-10-04;08:00;C/C++ programmation"
Event added!
$>
Copy

Adding an event on the same date/time is not an error.

Events are saved in current directory in a file named .calendar_datas (if file does not exists: no errors).

Events can also be removed with --delete argument:

$> ./calendar --delete "2018-10-04;08:00"
Event removed!
$> ./calendar --delete "2018-10-04;08:00"
Event not found!
$>
Copy

Usage of temporarys files is authorized (but don't forget to delete them at the end of your script!).
